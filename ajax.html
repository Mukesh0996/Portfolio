What is AJAX?

Introduction

AJAX stands for Asynchronous JavaScript and XML. AJAX is not a programming language, it is a concept in web development. It is a concept which uses a combination of a browser's built-in XMLHttpRequest object(used to send HTTP requests to the server) and JavaScript and HTML DOM.

How AJAX works?

As and when an event is fired in the UI(web page), we create an XMLHttPRequest object in the Javascript file which is wired to the web page and we send the HTTP Request over the network(Internet) to the server. Server is responsible for accepting(if certain conditions are met) and will process the HTTP Request. Server later creates a response and sends the data back to the browser through the internet. The data is received to the JavaScript which is running in the browser and the data(recevied from the server) is later updated in the HTML DOM.

How AJAX is implemented in the recent web apps?

AJAX is implemented using the FETCH API as XMLHttps is already in the depreciation state. The Fetch API is more powerful and is flexible. Fetch API is similar to that of the XMLHttpRequest object. Fetch API takes two arguments. 1 - Request URL(URL which points to the server) 2 - Request Options. Request options is an Object which consists of a few key value pairs. Some of them are:

1) Method - The method includes 4 values - GET (to fetch data from server), POST (to add data in the Database through the server), PUT (to update the data which is already present), DELETE (to delete the data in the server).

2) Headers - This includes the type of data that is being sent through the network. Most used header is "Content-Type": "application/json", when this is used, we indicate that the network request constains on JSON object.

3) Body - This consits of the data that is being sent through the network to the server. It is later accessed by the server to add/ process/ update/ delete the data.

Advantages

- AJAX allows web pages to update asynchronously by exchanging data with the server behind the scenes. It is possible to update parts of a web page, without even reloading the web page.

- It helps in enhancing the user experience as the web page doesn't reload for every network request.
